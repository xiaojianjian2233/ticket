## 二、页面清单

### 2.1 工单列表 `/tickets`
- **功能**：工单全量列表 + 筛选搜索
- **接口**：GET /tickets
- **表格列**：工单号/来源(标签)/标题/产品线/功能模块/责任人(空=未分配橙)/状态(标签)/退回(×N红)/创建时间/SLA(超时红)/操作
- **筛选**：来源/状态/产品线/责任人/是否退回/时间范围 + 关键词
- **按钮权限**：查看(全部)；转人工/关闭/改责任人(handler/admin)；重新入队(admin)；导出(handler/admin)
- **状态枚举**：见公共规则 3.4

### 2.2 未接管工单 `/tickets/unhandled`
- **功能**：status=returned 工单单独视图
- **接口**：GET /tickets/unhandled
- **表格列**：工单号/来源/标题/不接管原因(route_action)/时间/操作
- **按钮权限**：查看(全部)；手动改判+重新入队(admin)

### 2.3 工单详情 `/tickets/:id`
- **功能**：工单全生命周期展示
- **接口**：GET /tickets/{id}、/route-evidence、/handle、/close、/tag
- **区块**：基本信息 / 流转判断(结果+原因+evidence) / 打标(标签+置信度+历史+人工修正) / 去重(复用来源) / 答复(分支+final_reply+去AI化对比) / 研发进展(镜像) / SLA / 附件
- **按钮权限**：转人工/关闭/改标签(handler/admin)；改判/重入队/查raw(admin)；evidence(handler/admin)
- **不展示**：org/raw_json/PII

### 2.4 我的待办工作台 `/workbench`
- **功能**：B/C 派到名下的待处理工单
- **接口**：GET /workbench/my-tickets、POST /tickets/{id}/handle
- **表格列**：工单号/分支/标题/full_reply摘要/派单时间/SLA/操作
- **按钮权限**：确认AI答复关单/修改后回写/转研发/补充资料(handler/admin)

### 2.5 研发单列表 `/hubs`
- **功能**：研发待修 bug 列表
- **接口**：GET /hubs
- **表格列**：hub号/标题/产品线/责任人/关联工单数/Linear状态/研发状态/SLA-2/创建时间/操作
- **按钮权限**：查看/打开Linear(handler/admin)；重新同步(admin)
- **状态枚举**：linear_sync(pending/synced/failed)；研发(in_progress/resolved/closed/其它原样)

### 2.6 研发单详情 `/hubs/:id`
- **功能**：研发单 + 关联工单 + Linear 回调信息
- **接口**：GET /hubs/{id}、/resync-linear
- **区块**：基本信息/Linear信息/回调信息(处理人/说明/时间)/关联工单列表/SLA-2

### 2.7 FAQ 管理 `/faq`
- **功能**：FAQ 列表管理
- **接口**：GET /faq、PUT/DELETE /faq/{id}
- **表格列**：faq号/标题/产品线/审核状态(标签)/命中次数/来源工单/创建时间/操作
- **按钮权限**：查看(全部)；编辑/删除(admin)；审核历史(全部)
- **状态枚举**：pending_review/approved/rejected

### 2.8 知识库查阅 `/faq/browse`
- **功能**：语义搜索 + 浏览(默认 approved)
- **接口**：POST /faq/search、GET /faq/browse
- **视图**：卡片(title/摘要/产品线/命中次数/来源工单) + 顶部语义搜索框 + 产品线筛选
- **交互**：搜索显相似度%；引用按钮(填入回复框)

### 2.9 FAQ 审核 `/faq/review`
- **功能**：待审/驳回 FAQ 人工审核
- **接口**：GET /faq/review-list、POST /faq/{id}/review、PUT /faq/{id}
- **列表**：pending_review+rejected(红置顶)
- **审核详情**：title/content + 四维结果(✓/✗) + reject_dims + reason + 审核历史
- **按钮权限**：通过/驳回/编辑/删除(审核人/admin)
- **表单**：title(≤20校验)/content(≤300校验)/product_tag/图片

### 2.10 SLA 监控 `/sla`
- **功能**：双 SLA 超时监控 + 系统告警
- **接口**：GET /sla、/sla/overview、/sla/notify-log、/sla/system-alerts
- **概览卡片**：超时总数/今日新增/责任人top
- **表格列**：工单号/责任人/超时时长/SLA类型(人工/研发)/状态/超时天数
- **tab**：SLA列表 / 系统告警 / 通报历史
- **筛选**：SLA类型/责任人/产品线/超时区间
- **状态枚举**：normal(绿)/breached(红)

### 2.11 智能助手 `/assistant`
- **功能**：对话式 NL2SQL 统计 + 提单
- **接口**：POST /assistant/chat、/assistant/submit
- **交互**：聊天消息流 + 输入框；统计结果表格/图表；SQL(admin可见)；PII角色脱敏；提单确认卡片；写指令降级提示
- **按钮权限**：全部可用(只读统计)

### 2.12 系统配置页（admin）
| 页面 | 接口 | 核心 |
|---|---|---|
| 用户管理 `/system/users` | GET /users、PUT /users/{id} | 列表 + 改角色/启用禁用 |
| Skill 管理 `/system/skills` | GET /skills、PUT/rollback/preview | Markdown编辑器(9 LLM)/py只读 + 版本diff回滚 + 样例试跑 |
| 派单配置 `/system/dispatch` | /dispatch/assignees | 配额名单(主/溢出)+默认人 |
| 模块映射 `/system/module-owners` | /module-owners | 产品线/模块/触发词/责任人(精确+业务域兜底) |
| 系统告警 `/system/alerts` | /sla/system-alerts、requeue | 队列/依赖/abandoned + 重入队 |

## 三、公共规则

### 3.1 全局状态码映射（业务 code）
| code | 含义 | 前端处理 |
|---|---|---|
| 0 | 成功 | 正常 |
| 401 | 未登录/token过期 | 清token+跳/login+"登录已过期" |
| 403 | 无权限/账号禁用 | "无权限"提示 |
| 422 | 参数校验失败 | 表单字段错误提示 |
| 500 | 服务错误 | "系统繁忙，请重试" |
| 502 | 第三方失败 | "外部服务异常" |

### 3.2 提示文案（统一）
| 场景 | 文案 |
|---|---|
| 登录过期 | 登录已过期，请重新登录 |
| 无权限 | 您没有权限执行此操作 |
| AI处理中 | AI 处理中… |
| agent答复中 | AI答复生成中（可能需1-3分钟） |
| 流转判断中 | 流转判断中… |
| 依赖挂起 | 等待AI服务恢复 |
| 不接管 | 当前工单不接管（{原因}） |
| FAQ命中 | 命中知识库自动答复 |
| 复用历史 | 本工单复用 #{id} 答复自动关单 |
| 助手SQL拦截 | 仅支持数据查询，不支持修改操作 |
| 助手写指令 | 该操作暂未开放，请到工单页操作 |
| 助手无法理解 | 未能理解您的问题，可换种问法或举例 |
| FAQ驳回确认 | 驳回后该FAQ不再被检索 |
| 操作成功 | 操作成功 |
| Skill保存 | 已保存并热生效 |

### 3.3 权限控制规则
- **路由守卫**：无token→/login；admin路由非admin拦截。
- **按钮级**：visitor 隐藏所有写操作按钮（转人工/关闭/编辑/删除/配置）。
- **数据级**：PII 列(手机/邮箱/税号) visitor 脱敏(138****5678)，handler/admin 明文。
- **菜单级**：按 role 动态渲染。
- **审核人/SLA接收人**：独立配置名单（非 role 圈定）。

### 3.4 字典映射（状态枚举 → 标签 + 色）
**工单 status**：
| 值 | 标签 | 色 |
|---|---|---|
| pending | 待处理 | 灰 |
| returned | 已退回(不接管) | 红 |
| pending_manual | 待人工 | 橙 |
| pending_rd | 待研发 | 蓝 |
| done | 已处理 | 绿 |
| closed | 已关闭 | 深灰 |

**来源 source**：ksm(KSM蓝)/zhichi(智齿绿)/assistant(助手紫)
**分支 answer_branch**：A(bug红)/B(需求橙)/C(补料黄)/D(正常绿)
**流转 route_action**：可流转(绿)/不接管-多问题(红)/不接管-不可流转(红)
**FAQ review_status**：pending_review(待审灰)/approved(通过绿)/rejected(驳回红)
**Linear sync**：pending(同步中)/synced(已同步)/failed(失败红)
**研发 hub.status**：in_progress(处理中蓝)/resolved(已解决绿)/closed(已关闭灰)/其它(原样)
**SLA sla_state**：normal(正常绿)/breached(超时红)
**SLA type**：manual(人工)/rd(研发)
