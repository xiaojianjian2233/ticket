# ticket-hub 后端接口清单

> 编制：资深后端架构师 · 日期 2026-06-10
> 依据：业务流程(10项) + 数据库(14表) + 业务规则(10条) + 目录结构。
> 统一约定：返回体 `{code, message, data}`；鉴权 `Authorization: Bearer <JWT>`；分页 `data:{items,total,page,page_size}`。
> 权限：admin(全)/handler(业务读写)/visitor(只读)。

---

## 一、第三方对接接口（webhook，被动接收/回调）

### 1.1 KSM 工单推送
- **POST** `/webhook/ksm`
- 入参：`{noticeNum}`（+ webhook 鉴权头 WEBHOOK_ACCESS_TOKEN）
- 出参：`{code:0}`（立即返回，只入队）
- 用途：接收 KSM 推送 → 存 noticeNum → 入队 ksm_intake
- 异常：鉴权失败 401；报文非法记 integration_log 不入库
- 依赖：t_task_queue

### 1.2 智齿工单推送
- **POST** `/webhook/zhichi`
- 入参：完整工单 raw（+ 鉴权头）
- 出参：`{code:0}`
- 用途：接收智齿推送 → 入队 sync_ticket
- 异常：鉴权失败 401
- 依赖：t_task_queue

### 1.3 Linear 回调（研发状态同步）
- **POST** `/webhook/linear`
- 入参：`{issue_id, status, handler, note}`（状态/处理人/说明）
- 出参：`{code:0}`
- 用途：Linear 回写研发状态 → 按 status 分派（计划+日期→自动答复关单/计划无日期→转回/产研退回非空→转人工/退回空→转回/其它→展示）→ 镜像同步 info
- 异常：验签失败 401；hub 不存在记日志
- 依赖：t_ticket_hub, t_ticket_info, Linear(转回), 飞书

> 注：第三方**出站**调用（subscribeCallback/lockKsmOrder/handleKsmOrder/supplyKsmOrder/save_ticket_reply/ask agent/硅基流动/DeepSeek/Linear issueCreate/飞书）由 worker/service 内部发起，非对外 REST，见 integrations/。

## 二、工单业务接口

### 2.1 工单列表
- **GET** `/api/v1/tickets?source=&status=&product_tag=&dev_owner=&is_returned=&keyword=&page=&page_size=`
- 出参：`{items:[工单列表项], total, page, page_size}`
- 用途：工单列表（前端只展示 info 字段）
- 权限：全部（visitor 只读）
- 依赖：t_ticket_info

### 2.2 工单详情
- **GET** `/api/v1/tickets/{id}`
- 出参：工单完整信息（info 字段 + 标签历史 + 去重 + 答复 + SLA + 研发进展镜像；不含 org/raw_json/PII）
- 用途：详情页
- 权限：全部
- 依赖：t_ticket_info, t_ticket_tag, t_ticket_merge, t_attachment

### 2.3 未接管工单列表
- **GET** `/api/v1/tickets/unhandled?page=`
- 出参：`{items, total}`（status=returned）
- 用途：未接管工单单独视图
- 权限：全部
- 依赖：t_ticket_info

### 2.4 手动改判可流转 + 重新入队
- **POST** `/api/v1/tickets/{id}/rejudge`
- 入参：`{}`（可选 remark）
- 出参：`{code:0}`
- 用途：AI 误判不接管时纠正，重跑流水线
- 异常：非 returned 状态拒绝
- 权限：**admin**
- 依赖：t_task_queue, t_operation_log

### 2.5 手动重新拉取/重新入队（运维）
- **POST** `/api/v1/tickets/{id}/requeue`
- 入参：`{task_type}`（intake/run_pipeline）
- 出参：`{code:0}`
- 用途：处理 abandoned/拉取失败工单
- 权限：**admin**
- 依赖：t_task_queue

### 2.6 人工修正打标
- **PUT** `/api/v1/tickets/{id}/tag`
- 入参：`{product_tag, func_module, dev_owner}`
- 出参：`{code:0}`
- 用途：handler 改标签（插 t_ticket_tag manual 行 + 刷主表冗余值）
- 权限：admin/handler
- 依赖：t_ticket_tag, t_ticket_info, t_operation_log

### 2.7 工单流转判定依据
- **GET** `/api/v1/tickets/{id}/route-evidence`
- 出参：`{route_action, route_reason, evidence}`
- 用途：查看 LLM 流转判定理由
- 权限：admin/handler
- 依赖：t_skill_log

### 2.8 handler 工作台（我的待办）
- **GET** `/api/v1/workbench/my-tickets?branch=&status=&page=`
- 出参：`{items}`（dispatch_assignee=当前用户的 B/C 工单 + full_reply）
- 用途：handler 处理 B/C 派单工单
- 权限：handler/admin
- 依赖：t_ticket_info

### 2.9 工单人工处理（确认答复/修改回写/转研发/补充资料）
- **POST** `/api/v1/tickets/{id}/handle`
- 入参：`{action: confirm_reply|edit_reply|to_rd|supply, reply_content?}`
- 出参：`{code:0}`
- 用途：handler 处理工单——确认AI答复关单/修改后回写/转研发(转A)/补充资料
- 异常：回写来源系统失败 → writeback_error + 重试
- 权限：admin/handler
- 依赖：t_ticket_info, integrations(KSM/智齿), 飞书

### 2.10 工单关闭（回写处理完成）
- **POST** `/api/v1/tickets/{id}/close`
- 入参：`{reply_content?}`
- 出参：`{code:0}`
- 用途：回写来源系统到处理完成(KSM4/智齿3) + 记 resolved_at（14天后观察期扫描自动 closed）
- 权限：admin/handler
- 依赖：t_ticket_info, integrations

### 2.11 研发单(hub)列表
- **GET** `/api/v1/hubs?product_tag=&dev_owner=&linear_status=&status=&page=`
- 出参：`{items}`（hub + 关联工单数）
- 用途：研发单管理页
- 权限：admin/handler
- 依赖：t_ticket_hub

### 2.12 研发单详情
- **GET** `/api/v1/hubs/{id}`
- 出参：hub 完整 + Linear 信息 + 回调信息 + 关联 info 列表 + SLA-2
- 权限：admin/handler
- 依赖：t_ticket_hub, t_ticket_info

### 2.13 研发单重新同步 Linear
- **POST** `/api/v1/hubs/{id}/resync-linear`
- 出参：`{code:0}`
- 用途：linear_sync_status=failed 时重新 issueCreate
- 权限：admin
- 依赖：Linear, t_ticket_hub

## 三、FAQ 管理接口

### 3.1 FAQ 列表
- **GET** `/api/v1/faq?product_tag=&review_status=&keyword=&page=`
- 出参：`{items}`（faq_no/title/产品线/审核状态/命中次数/来源工单）
- 权限：全部
- 依赖：t_faq

### 3.2 FAQ 详情
- **GET** `/api/v1/faq/{id}`
- 出参：FAQ 完整 + 审核历史
- 权限：全部
- 依赖：t_faq, t_faq_review

### 3.3 FAQ 语义检索（人工查阅/工单辅助）
- **POST** `/api/v1/faq/search`
- 入参：`{query, product_tag?}`
- 出参：`{items:[{faq, similarity}]}`（topN，过滤 rejected）
- 用途：handler 主动查 FAQ / 工单侧边栏推荐
- 异常：硅基流动挂 → 降级关键词 like 搜
- 权限：全部
- 依赖：硅基流动, t_faq

### 3.4 FAQ 浏览（查阅页）
- **GET** `/api/v1/faq/browse?product_tag=&page=`
- 出参：`{items}`（默认只 approved）
- 权限：全部
- 依赖：t_faq

### 3.5 待审核 FAQ 列表
- **GET** `/api/v1/faq/review-list?page=`
- 出参：`{items}`（pending_review + rejected，rejected 置顶）
- 权限：审核人(配置名单)/admin
- 依赖：t_faq, t_faq_review

### 3.6 人工审核 FAQ（通过/驳回）
- **POST** `/api/v1/faq/{id}/review`
- 入参：`{result: approved|rejected, reason?}`
- 出参：`{code:0}`
- 用途：人工覆盖 AI 审核结果（最终决定权）
- 权限：审核人/admin
- 依赖：t_faq, t_faq_review, t_operation_log

### 3.7 编辑 FAQ
- **PUT** `/api/v1/faq/{id}`
- 入参：`{title(≤20), content(≤300), product_tag, images?}`
- 出参：`{code:0}`
- 用途：编辑 FAQ（编辑后直接 approved，不重新 AI 审）
- 异常：字数超限 422
- 权限：admin
- 依赖：t_faq（重算 embedding）

### 3.8 删除 FAQ（物理删）
- **DELETE** `/api/v1/faq/{id}`
- 出参：`{code:0}`
- 用途：物理删 FAQ + 连带删 t_faq_review
- 权限：admin
- 依赖：t_faq, t_faq_review

## 四、SLA 接口

### 4.1 SLA 监控列表
- **GET** `/api/v1/sla?sla_type=&owner=&product_tag=&overdue_range=&page=`
- 出参：`{items}`（合并人工/研发，sla_type 区分；工单号/责任人/超时时长/类型/状态/超时天数）
- 用途：SLA 监控页
- 权限：全部（全员可看全部）
- 依赖：t_ticket_info, t_ticket_hub, t_sla_log

### 4.2 SLA 概览统计
- **GET** `/api/v1/sla/overview`
- 出参：`{超时总数, 今日新增, 责任人top}`
- 权限：全部
- 依赖：t_sla_log

### 4.3 SLA 通报历史
- **GET** `/api/v1/sla/notify-log?date=&owner=&page=`
- 出参：`{items}`（t_sla_log 通报记录）
- 权限：全部
- 依赖：t_sla_log

### 4.4 系统告警列表（队列/依赖/abandoned）
- **GET** `/api/v1/sla/system-alerts?page=`
- 出参：`{items}`（队列积压/依赖挂起/abandoned）
- 用途：系统监控 tab
- 权限：admin
- 依赖：t_task_queue

### 4.5 abandoned 任务重入队
- **POST** `/api/v1/sla/requeue-abandoned/{task_id}`
- 出参：`{code:0}`
- 权限：admin
- 依赖：t_task_queue

> SLA 扫描器(每日08:00)/观察期扫描/队列监控由 scheduler 进程定时跑，非 REST 接口。

## 五、智能助手接口

### 5.1 助手对话（统计/提单）
- **POST** `/api/v1/assistant/chat`
- 入参：`{nl_query, session_id?}`
- 出参：`{type: query|submit|reject, sql?(admin可见), result?(表格/图表数据), explain, ticket_no?(提单时)}`
- 用途：NL2SQL 统计 / 对话提单
- 异常：NL2SQL 失败"换种问法"；guard 拦截"仅支持查询"；写指令"该操作暂未开放"
- 权限：全部（PII 按角色脱敏）
- 依赖：assistant-nl2sql SKILL, sql_guard(py), 硅基流动/LLM, t_assistant_log

### 5.2 对话提单确认
- **POST** `/api/v1/assistant/submit`
- 入参：`{title, description}`
- 出参：`{code:0, ticket_no}`
- 用途：确认卡片后建单（source=assistant，入流水线）
- 异常：校验失败（非空/≥15字/防重复）提示
- 权限：全部
- 依赖：t_ticket_info, t_task_queue

### 5.3 助手会话历史
- **GET** `/api/v1/assistant/sessions?session_id=&page=`
- 出参：`{items}`（多轮对话）
- 权限：本人/admin
- 依赖：t_assistant_log

## 六、登录 & 权限接口

### 6.1 飞书登录 URL
- **GET** `/api/v1/auth/feishu/login`
- 出参：`{authorize_url}`
- 用途：返回飞书授权 URL
- 依赖：飞书 OAuth

### 6.2 飞书回调
- **GET** `/api/v1/auth/feishu/callback?code=`
- 出参：`{access_token(JWT), expires_in, user}`
- 用途：code 换 JWT + 同步 t_users
- 异常：飞书失败 502；账号禁用 403
- 依赖：飞书 OAuth, t_users

### 6.3 当前用户信息
- **GET** `/api/v1/auth/me`
- 出参：`{id, feishuUid, name, email, role, avatarUrl}`
- 权限：登录态
- 依赖：t_users

### 6.4 用户列表
- **GET** `/api/v1/users?role=&keyword=&page=`
- 出参：`{items}`
- 权限：admin
- 依赖：t_users

### 6.5 修改用户角色/启用状态
- **PUT** `/api/v1/users/{id}`
- 入参：`{role?, is_active?}`
- 出参：`{code:0}`
- 用途：提权/禁用
- 权限：admin
- 依赖：t_users, t_operation_log

### 6.6 Skill 管理列表
- **GET** `/api/v1/skills`
- 出参：`{items}`（name/类型/version/更新人）
- 权限：admin
- 依赖：t_skill_md

### 6.7 取/编辑/回滚 Skill
- **GET/PUT** `/api/v1/skills/{name}`、**POST** `/api/v1/skills/{name}/rollback`、**POST** `/api/v1/skills/{name}/preview`
- 入参(PUT)：`{content_md}`；(preview)：`{sample_ticket}`
- 出参：SKILL.md 正文 / 版本 / 试跑结果
- 用途：编辑 9 个 LLM SKILL（校验 frontmatter→存→bump version→清缓存热生效）
- 权限：admin
- 依赖：t_skill_md, t_skill_md_history, t_operation_log

### 6.8 派单配额名单管理
- **GET/POST/PUT/DELETE** `/api/v1/dispatch/assignees`
- 入参：`{assignee_name, feishu_uid, alloc_value, tier(main|overflow)}`
- 出参：`{items}` / `{code:0}`
- 用途：配置派单配额名单 + 默认人
- 权限：admin
- 依赖：t_dispatch_assignee, t_dispatch_config

### 6.9 模块责任人映射管理
- **GET/POST/PUT/DELETE** `/api/v1/module-owners`
- 入参：`{product_tag, func_module, row_type, trigger_words, dev_owner, dev_owner_uid}`
- 出参：`{items}` / `{code:0}`
- 用途：维护打标映射（精确 + 业务域兜底）
- 权限：admin
- 依赖：t_module_owner, t_operation_log
