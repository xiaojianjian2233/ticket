# ticket-hub 接口清单

> 自动核对自 `app.main`（FastAPI 路由自省）+ `app/integrations/*`。统一前缀 `/api/v1`（webhook 与运维除外）。
> 鉴权：业务接口 Feishu SSO Bearer Token + RBAC（visitor/handler/admin）；webhook 走 `verify_webhook_token`。
> 对外写操作（KSM/智齿/Linear/飞书）经 `WRITEBACK_DRY_RUN` 守卫，开发期只组装不真发。

## 一、入站接口（我们对外提供，共 55 条）

| 方法 | 路径 | 说明 |

## 二、出站接口（我们调用第三方）

> 全部经各 client 的 dry_run 守卫；KSM/Linear 为**生产环境**。

### KSM（生产，`KSM_BASE_URL`，重试 ≤ `ksm_max_retry`=3）
| 接口 | 方法封装 | 用途 |
|------|----------|------|
| `/ierp/api/getAppToken.do` → `login.do` → access_token | `_access_token` | 三步鉴权 |
| `/ierp/kapi/app/open/subscribeCallback` | `subscribe_callback` | 凭单号拉全量工单 |
| `/ierp/kapi/v2/kded/kded_wos/lockKsmOrder` | `lock_order` | 接管工单 |
| `/ierp/kapi/v2/kded/kded_wos/handleKsmOrder` | `handle_order` | 答复并关单（D/dedup/人工/Linear自动） |
| `/ierp/kapi/v2/kded/kded_wos/supplyKsmOrder` | `supply_order` | **C 资料缺失：退回客户补充资料** |
| `/ierp/kapi/v2/kded/kded_wos/returnKsmOrder` | `return_order` | 人工退回工单 |

### 智齿（`ZHICHI_BASE_URL`）

## 三、内部任务队列（PG `t_task_queue`，非 HTTP）
| 任务类型 | 触发 | 处理 |
|----------|------|------|
| `ksm_intake` | KSM webhook | subscribeCallback 拉全量 → 建单 → run_pipeline |
| `sync_ticket` | 智齿 webhook | 归一 raw → 建单 → run_pipeline |
| `run_pipeline` | 建单后 | 跑双源流水线（见 `pipeline-v2-changes.md`） |
