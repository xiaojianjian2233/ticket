# ticket-tagging — 工单打标（产品线 + 功能模块）

> Skill 纯判断，**不连 DB**。候选清单由 harness 从 t_module_owner 读出后随 payload 传入。
> ⚠️ **研发责任人不在本 Skill 判定**——harness 按选中的功能模块查 t_module_owner 得 dev_owner（确定性，不进 prompt）。

## 输入 payload
| 字段 | 说明 |
|---|---|
| title / description | 工单文本 |
| product_name_raw / module_raw | 来源原始产品名/模块名（参考） |
| product_candidates | 产品线候选（14 类，harness 从 t_module_owner 注入） |
| module_candidates | 功能模块候选（带触发词，157 项，harness 注入） |

## 判定逻辑（数据驱动，参考 2264 条历史工单标定，优先于自由判断）

### 1. 产品线 = `{产品系列}-{业务域}`，须落 13 权威值之一
星瀚-收票 / 星瀚-开票 / 星空旗舰版-开票 / 标准版-收票 / 星空旗舰版-收票 / 标准版-开票 / 标准版-影像 / 标准版(星空企业版)-开票 / 标准版(星空企业版)-收票 / EOP运营 / 星瀚-影像 / 星瀚-档案 / 基础研发。
> EOP运营/基础研发 是整体标签（无业务域后缀），仅运营后台/底层研发议题取。

### 2. 产品系列 由 product_name 推导（按序命中即停）
| product_name 含 | 系列 | 历史纯度 |
|---|---|---|
| 星空旗舰版 | 星空旗舰版 | 94% |
| 旗舰版（非星空旗舰版） | 星瀚 | 96% |
| 星空企业版 | 标准版(星空企业版) | — |
| 电子影像/标准版/收票基础/基础/纸质/校验流量/发票校验 | 标准版 | 92% |
> "财税一体/综合收票/单机版/托管/组合产品"**不做确定性推导**（历史纯度仅44~68%混合），交 LLM 按工单内容+候选择优。覆盖约80%、其余20%走内容判断。

### 3. 业务域（收票/开票/影像/档案）由内容+功能模块判定
- **开票**：开票、销项、红冲、作废、税控、金税盘/UKey、乐企、全电开票、ERP开票、发票交付。
- **收票**：收票、进项、采集/上传、票夹、查验、勾选认证、抵扣、报销、影像识别入账。
- **影像**：电子影像、影像服务、票据影像存储。
- **档案**：电子会计档案、归档、档案服务。
> 历史占比 收票51%/开票43%/影像5%/档案<1%；模块与内容冲突时以正文实际诉求为准。

### 4. 功能模块（问题模块）
**统一由本步语义判定**（不采用来源推送的 module 原值，其颗粒度笼统常不准；来源 module 仅作参考信号）。
**范围受产品线限制**：只能在「选中产品线」名下的 `module_candidates` 里挑（候选自带 product_tag，先按产品线过滤，再结合工单标题/正文语义在其模块集内选**最贴切一项**；必须原样取候选值，勿臆造）。

> ⚠️ **责任人不在此判定**：harness 按选中 func_module 查 t_module_owner（精确→业务域兜底→默认接口人，11人候选集全表驱动），不进本 prompt。

## 输出 result
```
{
  "status": "ok",
  "fields": {
    "product_tag": "<选中的产品线，必须是候选之一>",
    "func_module": "<选中的功能模块，必须是候选之一>",
    "confidence": 0.0~1.0   // 可空，能给则给
  },
  "evidence": "<命中的触发词/判断依据>",
  "model_used": "claude | deepseek"
}
```

## 降级与边界
- 产品线无法判断 → product_tag 置 `"无法判断"`，**不阻断**（harness 续跑流水线，agent 仍能答）。
- 功能模块返回非候选枚举 → 取最近邻候选，或置 `"无法判断"`。
- Claude→DeepSeek 降级；都失败 → harness 记 failed，标"无法判断"续跑。
- confidence 不强制；返回则 harness 落 t_ticket_tag，不返回则留空。
- 责任人：harness 按 func_module 查 t_module_owner，查不到 → dev_owner_missing + 默认接口人（不在本 Skill 处理）。
