# ticket-routable — 工单流转判断（不拆单）

> Skill 纯判断，**不连 DB**。harness 传入 payload，本 Skill 输出 result 信封。

## 输入 payload
| 字段 | 说明 |
|---|---|
| title | 工单标题 |
| description | 问题描述 |
| product_name | 来源产品名（原始） |
| module | 来源模块名（原始） |
| has_attachment | 是否有附件(bool) |

## 判定规则（按序，命中即停）

按以下顺序判断，命中任一即输出对应结果，不再往下：

**① 非一对一（多问题）→ `不接管-多问题`**
满足任一：
- 文本含 ≥2 个显式问题编号（如 "1." "2."、"①②"、"问题一/问题二"）；
- 出现分隔语（"另外"、"还有"、"第二个问题"等）后接 **≥10 字** 的新问题描述。

**② 信息不足 → `不接管-不可流转`**
`description` 去除空白后 **< 15 字** **且** `has_attachment=false` **且** 语义上无法判断客户要解决什么（语义不明）。
> 三条件同时满足才判；描述短但语义清楚的不判。

**③ 非发票云模块 → `不接管-不可流转`**
`module` 命中**非发票云模块黑名单** **且** 全文（title+description）不含任何发票云关键词。

**④ 完全无发票云关键词 → `不接管-不可流转`**
全文（title+description）完全不含任何发票云关键词。

**⑤ 否则 → `可流转`**

## 清单（可页面热改）

**发票云关键词**（含任一即视为发票云相关；可页面增删）：
发票、发票云、开票、收票、影像、增值税、税控、金税、乐企、全电、进项、销项、税局、电子税局、通道、商户运营平台、纸电一体、OCR识别、查验、票池、红冲、抵扣、勾选、数电、版式、归档、税收分类编码
> （参考历史标定的完整清单；业务可继续增删，如：专票、普票、蓝字、作废、认证…）

**非发票云模块黑名单**（可页面增删）：
应收、应付、总账、生产制造、工资、薪酬、考勤、HR、人力资源、供应链、仓存、采购订单
> 注：来源模块名（module）若是 KSM 标准模块（如"开票管理""收票管理"）含强发票云信号，应纳入关键词命中判断，避免口语化描述（如"开不了票"）被误退回。

## 降级与边界
- Claude 失败 → DeepSeek；两者皆失败 → harness 将任务挂起等待（不在此 Skill 兜底，不猜测、不误放）。
- 退回单（is_returned）仍跑本判断；不接管按结果处理，可流转则续跑后续、末端由 harness 强制转人工。
- 本 Skill 只判断不执行动作；不接管后的来源系统动作（KSM 零回写/智齿话术关单/assistant 私信）由 harness 按来源处理。
